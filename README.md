# vue3-rbac-admin
第一个基础vue3项目
在本次后台管理前端项目的开发过程中，我基于 Vue3、Vite、Vue Router、Pinia、Element Plus 等技术栈，完成了一个具备基础企业级功能的 RBAC 权限管理系统。通过本项目的实践，我不仅加深了对前端工程化开发流程的理解，也进一步提升了组件化开发、路由管理、权限控制以及接口联调等方面的能力。

项目整体采用模块化与组件化的开发思想，将页面拆分为多个独立组件，提高了代码的复用性与可维护性。在开发过程中，我学习并掌握了 Vue3 Composition API 的基本使用方式，例如 ref、reactive、computed、watch 等核心 API，并能够利用其实现页面数据响应式更新。同时，通过使用 Vite 进行项目构建，我也了解了现代前端项目的启动流程、模块热更新以及打包优化机制。

在路由部分，项目使用 Vue Router 实现页面跳转与动态路由加载，并结合路由守卫实现登录鉴权功能。用户登录后，系统会将 token 存储到 localStorage 中，在页面刷新时依然能够保持登录状态。针对不同角色用户，系统能够动态生成对应菜单与访问权限，从而实现 RBAC 权限控制功能。这让我更加深入地理解了前后端权限分离的开发模式，以及动态路由和菜单渲染的实际应用场景。

在状态管理方面，我通过 Pinia 对用户信息、菜单权限以及系统状态进行统一管理，解决了组件之间数据共享的问题，也提高了代码结构的清晰度。与此同时，我还学习了接口封装、Axios 请求拦截器与响应拦截器的配置方式，实现了统一请求管理与错误处理机制。

在页面开发过程中，我使用 Element Plus 搭建后台管理界面，包括登录页、首页、用户管理、角色管理、菜单管理等常见后台模块。通过表格、表单、分页、弹窗等组件的使用，我进一步熟悉了企业后台系统常见的交互逻辑与页面布局方式。同时，我也逐渐掌握了前端项目中常见的目录结构设计与代码规范。

除此之外，本项目也让我认识到前端开发不仅仅是页面编写，更重要的是整体工程思维。例如项目初始化、环境配置、组件封装、权限设计、接口联调、异常处理以及项目优化等内容，都是完整项目开发中不可缺少的部分。在开发过程中，我也遇到了许多问题，例如路由匹配失败、store 初始化顺序错误、权限菜单无法渲染等，但通过不断查阅资料与调试代码，最终都得到了解决，这也提高了我独立分析问题与解决问题的能力。

总体来说，本次后台管理前端项目不仅巩固了我的 JavaScript、Vue 等前端基础知识，也让我对企业级前端项目的开发流程有了更加全面的认识。未来我还希望继续深入学习 TypeScript、前端性能优化、项目部署以及前后端协同开发等内容，不断提升自己的实际开发能力，为后续进入企业开发岗位打下更加扎实的基础。
